# Unity 2D Platformer Controller

A 2D platformer character controller built in Unity with C#. This project focuses on game feel, movement math, responsiveness, and the “juice” that makes a platformer controller feel good to play.

Fair warning: this controller may need some serious refactoring before being dropped into a polished production project. The structure is not perfect, but the core movement logic, tuning, and feel are there for building an awesome 2D platformer controller.

## Project Overview

This project was created as an experimental Unity 2D character controller focused on responsive player movement. The goal was not to build the cleanest architecture possible, but to explore how small movement details can make a platformer feel much better.

The controller is designed around fast iteration, movement tuning, and gameplay responsiveness. It includes the core math and feel needed for a strong 2D platformer foundation, even if the code itself could use cleanup and restructuring.

## Features

* 2D player movement built in Unity
* C# based controller logic
* Responsive platformer-style movement
* Tunable movement values for better game feel
* Focus on acceleration, control, and responsiveness
* Built as a foundation for future refactoring and expansion
* Good starting point for experimenting with platformer movement systems

## Why I Built It

I built this controller to better understand what makes 2D platformer movement feel good. A lot of platformers live or die based on small details like acceleration, jump feel, input timing, and responsiveness, so this project was focused on experimenting with those systems directly.

This was less about writing perfect production code and more about getting the movement to feel right first.

## Refactoring Note

This project is intentionally a little rough around the edges.

The movement logic and game-feel tuning are the main value of the project, but the code structure could be improved. Future refactoring could include:

* Separating movement logic into smaller scripts
* Cleaning up variable names and organization
* Improving input handling
* Adding clearer state management
* Creating reusable components
* Improving comments and documentation
* Making the controller easier to plug into other Unity projects

In other words: the bones are good, but the house needs renovations.
